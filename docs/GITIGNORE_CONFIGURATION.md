# .gitignore Configuration Guide

## Overview

This document explains the .gitignore configuration for the Product Search Agent project, ensuring sensitive files and build artifacts are not committed to version control.

## File Structure

```
product-search-agent/
├── .gitignore              # Root-level (covers both backend and frontend)
├── backend/
│   └── .gitignore         # Backend-specific ignores
└── frontend/
    └── .gitignore         # Frontend-specific ignores
```

## Root .gitignore

Located at: `/.gitignore`

**Purpose:** Covers common patterns for both backend and frontend

**Key Sections:**
- Python files (`__pycache__/`, `*.pyc`)
- Virtual environments (`venv/`, `.venv/`)
- Environment variables (`.env`, `*.env`)
- Node modules (`node_modules/`)
- Database files (`*.db`, `*.sqlite`)
- IDE files (`.vscode/`, `.idea/`)
- OS files (`.DS_Store`, `Thumbs.db`)
- Logs (`*.log`, `logs/`)
- Testing artifacts (`.coverage`, `.pytest_cache/`)

## Backend .gitignore

Located at: `/backend/.gitignore`

**Purpose:** Python/FastAPI specific ignores

### Key Categories

#### 1. Python Files
```gitignore
__pycache__/
*.py[cod]
*$py.class
*.so
.Python
```

**Why:** Compiled Python files should be regenerated, not committed.

#### 2. Virtual Environments
```gitignore
venv/
.venv/
env/
ENV/
```

**Why:** Virtual environments are large and machine-specific.

#### 3. Environment Variables & Secrets
```gitignore
.env
.env.local
.env.*.local
*.env
!.env.example
```

**Why:** Contains sensitive data (API keys, passwords, database URLs).

**Note:** `!.env.example` means `.env.example` IS committed (it's a template).

#### 4. Database Files
```gitignore
*.db
*.sqlite
*.sqlite3
*.db-journal
product_search.db
```

**Why:** Database files contain user data and can be large.

#### 5. Testing Artifacts
```gitignore
.pytest_cache/
.coverage
htmlcov/
.tox/
```

**Why:** Generated during testing, should be regenerated.

#### 6. Logs
```gitignore
*.log
logs/
pip-log.txt
```

**Why:** Logs can be large and contain sensitive information.

#### 7. Scraped Data & Cache
```gitignore
scraped_data/
cache/
*.cache
```

**Why:** Scraped data can be large and should be regenerated.

#### 8. Selenium Drivers
```gitignore
chromedriver
geckodriver
```

**Why:** Browser drivers are platform-specific and large.

## Frontend .gitignore

Located at: `/frontend/.gitignore`

**Purpose:** React/Vite/TypeScript specific ignores

### Key Categories

#### 1. Dependencies
```gitignore
node_modules/
jspm_packages/
bower_components/
```

**Why:** Dependencies are large (100+ MB) and should be installed via `npm install`.

#### 2. Build Output
```gitignore
dist/
dist-ssr/
build/
.vite/
.cache/
```

**Why:** Build artifacts are generated by `npm run build`.

#### 3. Environment Variables
```gitignore
.env
.env.local
.env.*.local
*.env
!.env.example
!.env.development
!.env.production
```

**Why:** Contains API URLs and secrets.

**Note:** `.env.development` and `.env.production` ARE committed (they're templates with placeholder URLs).

#### 4. Logs
```gitignore
logs/
*.log
npm-debug.log*
yarn-debug.log*
```

**Why:** Logs can be large and contain debugging information.

#### 5. Testing
```gitignore
coverage/
.nyc_output/
test-results/
playwright-report/
```

**Why:** Test results and coverage reports are generated.

#### 6. TypeScript
```gitignore
*.tsbuildinfo
.tsbuildinfo
```

**Why:** TypeScript build cache, should be regenerated.

#### 7. Package Manager
```gitignore
.yarn/cache
.pnp.*
.pnpm-debug.log*
```

**Why:** Package manager cache and lock files.

#### 8. Deployment Platforms
```gitignore
.vercel
.netlify/
.render/
```

**Why:** Platform-specific configuration and cache.

## Important Files to KEEP in Git

### Backend

✅ **DO commit:**
- `.env.example` - Template for environment variables
- `requirements.txt` - Python dependencies
- `Procfile` - Deployment configuration
- `*.py` - All Python source files
- `tests/` - All test files
- `docs/` - Documentation

❌ **DON'T commit:**
- `.env` - Contains secrets
- `venv/` - Virtual environment
- `*.db` - Database files
- `__pycache__/` - Compiled Python
- `*.log` - Log files

### Frontend

✅ **DO commit:**
- `.env.development` - Development template (with localhost URLs)
- `.env.production` - Production template (with placeholder URLs)
- `package.json` - Dependencies list
- `package-lock.json` - Dependency lock file
- `*.ts`, `*.tsx` - All TypeScript source files
- `*.css` - Stylesheets
- `public/` - Static assets
- `index.html` - Entry HTML

❌ **DON'T commit:**
- `.env` - Local overrides
- `.env.local` - Local secrets
- `node_modules/` - Dependencies
- `dist/` - Build output
- `*.log` - Log files

## Checking What's Ignored

### View ignored files
```bash
git status --ignored
```

### Check if a specific file is ignored
```bash
git check-ignore -v path/to/file
```

### List all ignored files
```bash
git ls-files --others --ignored --exclude-standard
```

## Common Scenarios

### Scenario 1: Accidentally Committed .env

**Problem:** You committed `.env` with secrets.

**Solution:**
```bash
# Remove from Git but keep locally
git rm --cached .env

# Commit the removal
git commit -m "Remove .env from version control"

# Push changes
git push
```

**Important:** Change all secrets that were exposed!

### Scenario 2: Accidentally Committed node_modules/

**Problem:** You committed the entire `node_modules/` folder.

**Solution:**
```bash
# Remove from Git but keep locally
git rm -r --cached node_modules/

# Commit the removal
git commit -m "Remove node_modules from version control"

# Push changes
git push
```

### Scenario 3: Accidentally Committed Database

**Problem:** You committed `product_search.db`.

**Solution:**
```bash
# Remove from Git but keep locally
git rm --cached backend/product_search.db

# Commit the removal
git commit -m "Remove database from version control"

# Push changes
git push
```

### Scenario 4: Want to Commit a Normally Ignored File

**Problem:** You want to commit a specific `.env` file for documentation.

**Solution:**
```bash
# Force add the file
git add -f path/to/.env

# Or update .gitignore to allow it
echo "!path/to/.env" >> .gitignore
```

## Best Practices

### ✅ DO:

1. **Review .gitignore before first commit:**
   ```bash
   cat .gitignore
   cat backend/.gitignore
   cat frontend/.gitignore
   ```

2. **Use .env.example for templates:**
   ```bash
   # Copy and customize
   cp .env.example .env
   # Edit .env with your actual values
   ```

3. **Check status before committing:**
   ```bash
   git status
   git status --ignored
   ```

4. **Add patterns as you discover them:**
   ```bash
   echo "new-pattern/" >> .gitignore
   ```

5. **Commit .gitignore changes:**
   ```bash
   git add .gitignore
   git commit -m "Update .gitignore"
   ```

### ❌ DON'T:

1. **Don't commit secrets:**
   - API keys
   - Passwords
   - Database credentials
   - OAuth tokens

2. **Don't commit large files:**
   - Database files
   - node_modules/
   - venv/
   - Build artifacts

3. **Don't commit generated files:**
   - `__pycache__/`
   - `dist/`
   - `*.pyc`
   - Coverage reports

4. **Don't commit IDE settings:**
   - `.vscode/` (except shared settings)
   - `.idea/`
   - `*.swp`

5. **Don't commit logs:**
   - `*.log`
   - `logs/`
   - Debug files

## Security Checklist

Before pushing to GitHub:

- [ ] `.env` is in .gitignore
- [ ] No API keys in code
- [ ] No passwords in code
- [ ] No database files committed
- [ ] `.env.example` has placeholder values only
- [ ] Secrets are documented in README (how to obtain them)

## Troubleshooting

### Issue: File still tracked after adding to .gitignore

**Cause:** File was already tracked before adding to .gitignore.

**Solution:**
```bash
# Remove from Git tracking
git rm --cached path/to/file

# Commit the change
git commit -m "Stop tracking file"
```

### Issue: .gitignore not working

**Cause:** Git cache needs to be cleared.

**Solution:**
```bash
# Remove all files from Git cache
git rm -r --cached .

# Re-add all files
git add .

# Commit
git commit -m "Fix .gitignore"
```

### Issue: Want to see what would be ignored

**Solution:**
```bash
# Dry run
git add --dry-run --all
```

## Platform-Specific Notes

### Render.com

Render.com automatically ignores:
- `.git/`
- `node_modules/` (reinstalled during build)
- `venv/` (recreated during build)

You still need .gitignore for local development and GitHub.

### GitHub

GitHub has size limits:
- File size: 100 MB max
- Repository size: 1 GB recommended max

Use .gitignore to avoid hitting these limits.
